# ①課題名
じゃんけんスロットゲーム

## ②課題内容（どんな作品か）
- スロットマシンとじゃんけんを組み合わせた対戦型ゲーム
- プレイヤーとコンピューターが3つのスロットを回し、揃った手の数で点数を競う
- 3つ揃い=3点、2つ揃い=2点、バラバラ=1点で勝敗を判定
- Web Audio APIを使用した効果音付き（スタート音、停止音、勝利音、負け音、あいこ音）
- スロットの停止順がランダムに変わる演出

## ③アプリのデプロイURL
なし

## ④アプリのログイン用IDまたはPassword（ある場合）
- なし

## ⑤工夫した点・こだわった点
- 5種類の効果音を1つの関数（`playSound`）で管理し、コードを簡潔にしたこと
- UIの変更
- 4パターンの停止順（左→中→右、右→中→左など）をランダムに選択し、毎回異なる演出を実現
- プレイヤーとコンピューターのスロット処理を1つの関数で処理
- ゲーム状態を管理し、回転中のボタン連打を防止機能を追加した。結果表示のフェードイン効果で臨場感を演出
- 各セクションに説明を追加（コメントで）

## ⑥難しかった点・次回トライしたいこと（又は機能）
- Web Audio APIの音声制御で、複数の音を順番に鳴らすタイミング調整や、フェードアウト効果の実装に試行錯誤が必要だった
- 3つのスロットが個別に停止するタイミングで、全体の状態を正しく管理する必要があった
- **次回トライしたいこと**:
  - スコアの累積機能（複数ラウンドの勝敗記録）
  - アニメーション効果の強化（勝利時の派手な演出など）
  - 難易度設定（コンピューターの強さ調整）
  - スマートフォン対応のレスポンシブデザイン
  - ローカルストレージを使った最高記録の保存
